# Robot Profiles

The `RobotProfile` pattern lets you swap robot configurations without changing any library code. A profile encapsulates two things: how to build the drive subsystem, and where the robot starts on the field.

---

## The three deployment scenarios

| Scenario | Profile used | How to trigger |
|----------|-------------|----------------|
| Library CI / unit dev | `SimRobotProfile` | `.\gradlew.bat test -PtestSim` |
| VSCode "Simulate Robot Code" | `ExampleRobotProfile` | `.\gradlew.bat simulateJava` (default) |
| Real hardware | `ExampleRobotProfile` | Deploy to RoboRIO |

`SwerveRobotContainer.selectProfile()` picks automatically:

```java
// Returns SimRobotProfile when -PtestSim is set.
// Otherwise reflectively instantiates the named class (ExampleRobotProfile by default):
if (Boolean.getBoolean("testSim")) return new SimRobotProfile();
return (RobotProfile) Class.forName(realProfileClassName)
    .getDeclaredConstructor().newInstance();
```

`ExampleRobot` passes the profile class name:

```java
public ExampleRobot() {
    super(SwerveRobotContainer.selectProfile("org.frc5010.examples.ExampleRobotProfile"));
}
```

---

## `SimRobotProfile` — library CI and development

`SimRobotProfile` uses small, square `ModuleType.SIM` / `GyroType.SIM` constants with no real CAN IDs. It is the lightweight profile for automated test agents and CI. Nothing needs to be changed here.

---

## `ExampleRobotProfile` — your robot

`src/main/java/org/frc5010/examples/ExampleRobotProfile.java` is the team's entry point. It already has the right structure; you only need to fill in real values.

### Step 1 — Fill in SwerveConstants

```java
import static edu.wpi.first.units.Units.*;

private static final SwerveConstants CONSTANTS = new SwerveConstants.Builder()
    .moduleType(ModuleType.TALON_FX)   // or SPARK_TALON, SPARK_MAX
    .gyroType(GyroType.PIGEON2)        // or NAVX
    .gyroCanId(1)

    .trackWidth(Inches.of(22.75))      // left-to-right wheel centres
    .wheelBase(Inches.of(22.75))       // front-to-back wheel centres
    .wheelRadius(Inches.of(2.0))       // actual worn radius

    .maxLinearSpeed(MetersPerSecond.of(4.5))
    .maxAngularSpeed(RadiansPerSecond.of(2 * Math.PI))

    .robotMass(Pounds.of(125))
    .bumperLength(Inches.of(30))
    .bumperWidth(Inches.of(30))

    .frontLeftIds(1, 2, 3)             // [drive, steer, encoder]
    .frontRightIds(4, 5, 6)
    .backLeftIds(7, 8, 9)
    .backRightIds(10, 11, 12)

    .canBusName("")                    // "" = RIO bus; CANivore name for Phoenix Pro
    .odometryFrequency(Hertz.of(100))  // 250 Hz for CANivore
    .build();
```

### Step 2 — Set the starting pose

```java
private static final Pose2d BLUE_START = new Pose2d(1.5, 5.5, new Rotation2d());
```

This is the Blue-alliance starting position for this year's game. `SwerveRobotContainer` automatically mirrors it to the Red side when `DriverStation.getAlliance()` returns `Red`.

### Step 3 — Wire hardware IO (TALON_FX)

`SwerveFactory.build()` throws `UnsupportedOperationException` for `TALON_FX` in REAL mode — it cannot construct motor configs without CTRE TunerX `SwerveModuleConstants`. Wire the real IO directly:

```java
@Override
public AkitSwerveDrive createDrive() {
    if (RobotBase.isReal()) {
        GyroIO gyro = new GyroIOPigeon2(CONSTANTS);
        ModuleIO[] modules = {
            new ModuleIOTalonFXReal(CONSTANTS, TunerConstants.FrontLeft),
            new ModuleIOTalonFXReal(CONSTANTS, TunerConstants.FrontRight),
            new ModuleIOTalonFXReal(CONSTANTS, TunerConstants.BackLeft),
            new ModuleIOTalonFXReal(CONSTANTS, TunerConstants.BackRight),
        };
        return new AkitSwerveDrive(CONSTANTS, gyro, modules);
    }
    // SIM: IronMaple uses the real robot's mass/geometry for accurate physics
    return SwerveFactory.build(CONSTANTS, BLUE_START);
}
```

`TunerConstants` is generated by **Phoenix Tuner X → Swerve Project Generator**. Each `TunerConstants.FrontLeft` etc. carries gear ratios, motor IDs, PID gains, and encoder offsets.

### Step 3 (alternate) — Wire hardware IO (SPARK_TALON)

For a NEO drive (SparkMAX) + Falcon/Kraken steer (TalonFX) robot:

```java
if (RobotBase.isReal()) {
    GyroIO gyro = new GyroIONavX();    // or GyroIOPigeon2(CONSTANTS)
    ModuleIO[] modules = {
        new ModuleIOSparkTalon(CONSTANTS, 0),  // 0=FL, 1=FR, 2=BL, 3=BR
        new ModuleIOSparkTalon(CONSTANTS, 1),
        new ModuleIOSparkTalon(CONSTANTS, 2),
        new ModuleIOSparkTalon(CONSTANTS, 3),
    };
    return new AkitSwerveDrive(CONSTANTS, gyro, modules);
}
```

## `SwerveRobotContainer` — what you get for free

`frc.robot.ExampleRobot` extends `SwerveRobotContainer`, which provides:

| Feature | Details |
|---------|---------|
| Keyboard drive | WASD on Glass joystick 0; axis 0=strafe, 1=forward, 2=rotate |
| Alliance-aware drive | Red alliance flips vx/vy so "forward" always faces the opponent wall |
| `resetToAllianceStart()` | Call from `autonomousInit()` and `teleopInit()` to teleport to starting pose |
| `getAutonomousCommand()` | Returns `SwerveVisualTest` when `-PvisualTest` is set; otherwise `null` |

### Customising bindings

`controller` is an `XboxConfigurableController` (protected field from `SwerveRobotContainer`). Add bindings by overriding `configureBindings()` in `ExampleRobot.java`:

```java
// In ExampleRobot.java
@Override
protected void configureBindings() {
    super.configureBindings();           // keeps keyboard drive on port 0
    // Add an extra binding on top of the default drive command:
    controller.button(1)
        .onTrue(Commands.runOnce(() -> drive.setPose(Pose2d.kZero)));
}
```

To replace the keyboard drive with an Xbox controller (typical for competition):

```java
// In ExampleRobot.java
@Override
protected void configureBindings() {
    // Do NOT call super — we're replacing the entire default command.
    XboxConfigurableController driver = new XboxConfigurableController(0);

    JoystickAxis forward  = driver.leftY().negate().deadzone(0.05).power(2.0);
    JoystickAxis strafe   = driver.leftX().negate().deadzone(0.05).power(2.0);
    JoystickAxis rotation = driver.rightX().negate().deadzone(0.10);
    DriveVector translate = DriveVector.of(forward, strafe).unitCircle();

    drive.setDefaultCommand(
        Commands.run(
            () -> {
              double flip = DriverStation.getAlliance().orElse(Alliance.Blue) == Alliance.Red
                  ? -1.0 : 1.0;
              Translation2d xy = translate.get();
              drive.runVelocityFieldRelative(new ChassisSpeeds(
                  flip * xy.getX() * drive.getMaxLinearSpeed().in(MetersPerSecond),
                  flip * xy.getY() * drive.getMaxLinearSpeed().in(MetersPerSecond),
                  rotation.getAsDouble() * drive.getMaxAngularSpeed().in(RadiansPerSecond)));
            },
            drive
        ).withName("XboxDrive")
    );

    driver.a().onTrue(Commands.runOnce(() -> drive.setPose(
        new Pose2d(drive.getPose().getTranslation(), new Rotation2d()))));
}
```

### Customising auto

Override `getAutonomousCommand()` in `ExampleRobot.java` to integrate PathPlanner, Choreo, etc.:

```java
// In ExampleRobot.java
@Override
public Command getAutonomousCommand() {
    if (Boolean.getBoolean("visualTest")) return SwerveVisualTest.build(drive, vision, this::getAllianceStartPose);
    return AutoBuilder.buildAuto("MyAuto");
}
```

## Field length

The default field length is **16.540988 m** (Arena2026Rebuilt). To override for a different year:

```java
// In your ExampleRobotProfile or RobotContainer subclass
@Override
public Distance getFieldLength() {
    return Meters.of(16.541);  // 2026 standard; adjust for other years
}
```
